SePay Webhook

Hướng dẫn kết nối thanh toán tự động

Làm theo các bước bên dưới để SePay gửi giao dịch về bot. Khi khách chuyển khoản đúng nội dung, backend sẽ nhận webhook tại /webhook/payment, xác thực API key, ghi log và tự xử lý đơn hàng hoặc yêu cầu nạp ví.

Chuẩn bị trước khi cấu hình

  • Server bot phải truy cập được từ Internet bằng domain hoặc IP public.
  • Biến cấu hình WEBHOOK_SECRET trên server phải trùng API Key nhập trong SePay.
  • Nếu chạy sau reverse proxy, hãy trỏ URL SePay tới domain HTTPS public thay vì URL nội bộ.
  • Sau khi lưu webhook, kiểm tra tab Webhook trong Admin để xem request có vào hệ thống không.
1
Mở mục Tích hợp Webhooks

Trong dashboard SePay, mở Quick menu rồi chọn Tích hợp Webhooks. Đây là nơi quản lý các webhook mà SePay sẽ gọi khi có giao dịch.

2
Thêm webhook mới

Ở màn hình Webhooks, bấm Thêm webhook. Nếu đã có webhook cũ, có thể giữ lại để đối chiếu nhưng chỉ nên bật webhook đang trỏ về server hiện tại.

3
Nhập thông tin cơ bản

Đặt tên dễ nhận biết, ví dụ bot. Ở ô URL nhận webhook, nhập URL public của server kèm path /webhook/payment. Chọn loại giao dịch Tiền vào và định dạng dữ liệu JSON.

URL theo domain hiện tại: /webhook/payment. Hãy copy đúng URL này vào SePay để webhook gọi về server đang chạy.
4
Chọn tài khoản ngân hàng

Chọn Tất cả tài khoản nếu mọi tài khoản đã liên kết đều dùng cho bot, hoặc chọn Tùy chọn nếu chỉ một tài khoản nhận tiền cho shop.

Nếu bật "Dùng để xác thực thanh toán", hãy chắc chắn tài khoản nhận tiền trong SePay trùng tài khoản đang cấu hình trong bot.
5
Cấu hình API Key

Chọn phương thức xác thực API Key. Giá trị API Key nhập vào SePay phải giống biến WEBHOOK_SECRET trên server bot.

Backend sẽ kiểm tra header SePay gửi lên: Authorization: <WEBHOOK_SECRET>. Nếu sai key, log webhook sẽ hiện Auth = no và request bị từ chối.
6
Hoàn tất và kiểm tra log

Ở bước cảnh báo, có thể bật hoặc tắt tùy nhu cầu vận hành, sau đó bấm Thêm. Tạo một giao dịch thử với đúng nội dung chuyển khoản để kiểm tra xử lý tự động.

Trong Admin, mở tab Webhook, lọc nhà cung cấp SePay. Request hợp lệ sẽ có Auth = yes; nếu không khớp đơn, kiểm tra lại nội dung chuyển khoản.
Lưu ý bảo mật: không chia sẻ WEBHOOK_SECRET. Khi nghi ngờ key đã lộ, đổi key trên server, cập nhật lại API Key trong SePay, rồi kiểm tra log webhook sau giao dịch thử.
